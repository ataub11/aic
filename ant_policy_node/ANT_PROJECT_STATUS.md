## Overview

A geometric cable-insertion policy for a UR5e robot arm in the AI for Industry Challenge. The robot must autonomously insert SFP or SC optical cables into board ports across 3 trials per run.

**Scoring rules:**
- tier_1: model validation (+1)
- tier_2: force/trajectory bonuses and penalties. Force sustained > 20 N absolute for ≥ 1 s = **−12 per trial**
- tier_3: physical insertion confirmed (+score if plug seated in port)

**Trial configurations (fixed across all runs):**
- Trial 1: SFP plug, board yaw = +17°
- Trial 2: SFP plug, board yaw = −45°
- Trial 3: SC plug, board yaw = +17°

## 4-Stage Pipeline Architecture

```
insert_cable()
  ├── Stage 1: _localize_connector()         — vision localisation → port XY
  │     SFP path: fast path to calibrated port (vision blind top-down)
  │     SC path:  2-point lateral pre-scan at transit_z=connector_z+0.18
  │       _detect_port_pixel_sc_prescan()    — centre-masked Otsu → housing blob (u,v)
  │       _backproject_to_base_xy()          — camera TF + CameraInfo K → base_link XY
  ├── Stage 2: _approach_connector()         — move to 10 cm above connector Z
  ├── Stage 3: _detect_contact()             — direct descent to connector_z (both SFP and SC)
  │     SFP (Bug 47) + SC (Bug 60): zero axial force → FTS blind → direct descent
  │     Generic FTS spiral retained in code for unknown connector types (unreachable in competition)
  ├── Stage 4: _compliant_insertion()        — descend to connector_z−5mm in 2mm steps
  └── On SurfaceContactError:
        _retreat_from_surface()              — ascend 15 cm at current XY (NO lateral move)
        retry _run_pipeline(settle_sec=1.5) once
```

## Stage 3 — Contact Detection (Direct Descent)

**Both SFP and SC use direct descent (Bugs 47 + 60):** Neither cable type generates detectable axial force at the port entrance. FTS-based contact detection is structurally blind for both. `_detect_contact` calls `_move_to_pose_and_wait(convergence_m=0.010, stage_timeout_sec=20s, check_force=False)` targeting `connector_z`. Returns actual TCP pose at stall depth; Stage 4 covers the remaining gap.

**Generic FTS spiral (retained in code, unreachable in competition):** For unknown connector types only. Uses running-minimum local baseline, 5mm/2mm adaptive step size, 49-offset spiral (±5–110 mm), near-port 1-of-1 shortcut at 4 N, and Z-proximity guard with surface detach. Not exercised since Run 20 (SFP) / Run 24 (SC).

## Stage 4 — Compliant Insertion

- **Bug 67 fix (2026-04-14):** Immediately holds cmd_z = `connector_z − 5mm` for up to 120 s. Constant sustained load accelerates viscoelastic cable creep. All 3 trials use constant hold (Bug 68 ramp removed 2026-04-17).
- **Bug 72 fix (2026-04-17):** `insertion_stiffness = 200 N/m` used for T1/T2 (SFP) in Stage 4. Combined with max_wrench=15N: T1 spring=15N (saturated), T2 spring=10.2N. Higher than old 8.8N → more force authority on real hardware.
- **Bug 74 fix (2026-04-17):** T3 (SC) Stage 4 uses `approach_stiffness = 85 N/m`. With 200N/m+15N ceiling, SC arm overshoots viable tcp_z zone (0.09–0.10m) → tcp_z=0.0304, tier_3=0 (Run 31). At 85N/m spring=7.7N < cable tension → slow creep → arm stays in viable zone → tier_3≈6–8.
- `approach_stiffness = 85 N/m` (used for all motion except T1/T2 Stage 4)
- `insertion_stiffness = 200 N/m` (T1/T2 Stage 4 only; `max_wrench=15 N` hard ceiling)
- **Exits:** force abort (solid surface → False), tcp_z ≤ end_z + 5mm (insertion depth → break→True), 120 s timeout (arm as deep as cable allows → break→True). All non-abort exits return True.
- **Force abort:** `force_abs > baseline + 3 N` sustained continuously > zone-budget → arm on solid surface, not in port. Timer resets to zero whenever force drops below threshold (Bug 90: measures continuous sustained force, not cumulative — cable oscillations don't accumulate abort budget). Budget: 0.5 s for SC, 0.3 s for SFP.
- **SFP Stage 4 feedforward_fz (Run 36+, escalated Run 40):** `feedforward_wrench_at_tip` carries an additional downward Z force, routing through the feedforward path (max 40N, bypassing max_wrench=15N cap). T1: −3N → total 15+3=18N (confirmed working on real hardware ECR v10, arm descended to end_z in 8.02s). T2: −9N → total 10.2+9=19.2N (−3N/18N insufficient; −6N/16.2N still frozen in Run 39; escalated to −9N). **SC feedforward_fz=−5N (Bug 90, ECR v12):** At stall the 85N/m spring (7.6N) ≈ SC cable tension (7.7N) → zero net force → arm frozen at tcp_z≈0.097m. The −5N feedforward provides 12.6N total downward → net 4.9N downward at stall → slow controlled descent toward viable depth. **Sim note:** MuJoCo cable plugin has no axial stiffness key → cable is inextensible; T1/T2 stall heights are cable-length limits, not force equilibria. Only real hardware validates Stage 4 feedforward.
- **After Stage 4:** arm stays at depth. No joint return for any trial (Bugs 52 + 61) — scoring captures arm at insertion depth.

## Surface Recovery

**Critical: ascend only, no lateral move.**

Moving arm to scouting XY during retreat → retry Stage 1 collects both rays at/near scouting XY → zero baseline → degenerate triangulation every time. Staying at (surface_x, surface_y, surface_z+0.15) guarantees the retry starts from a different position.

On retry: `baseline_settle_sec = 1.5 s` — waits for arm to stop decelerating before calibrating baseline. Without this: 7.64 N baseline observed (vs normal ≈ 22 N) → wrong contact thresholds.

---

## Cable Baseline Calibration

- 20 FTS samples at ~0.1 s intervals while arm is stationary
- **Median** (not mean) — SC reversed cable oscillates wildly (std = 28 N observed)
- **Std > 5 N rejection** — if noisy, keep previous baseline (arm may still be moving)
- `cable_force_baseline_default = 22.5 N` fallback
- Re-calibrated at start of Stage 3 (approach height force profile differs from pre-scout)

## Systematic Process Flaws (updated Run 8 — 2026-04-05)

These are structural failure modes that fire every trial, not one-offs.

---

### Process Flaw 1 — 0% triangulation success rate

Every triangulation attempt across Runs 7 and 8 failed:

| Run | Trial | Outcome | Reason |
|-----|-------|---------|--------|
| 7 | T1 | residual=0.056 m → rejected | |
| 7 | T2 | ΔZ=0.2 cm → discarded | arm stall (Bug 19, fixed) |
| 7 | T3 | residual=0.074 m → rejected | |
| 8 | T1 | residual=0.056 m → rejected | |
| 8 | T2 | residual=0.042 m → rejected | |
| 8 | T3 | negative depth (t1=−0.161, t2=−0.049) → failed | diverging rays |

**Root cause:** The CV detector finds different scene elements in each view — perspective shift on port face aperture corners. Rays point to different physical points → residual >> 0.02 m every time.

**Consequence:** Triangulation was replaced entirely with FTS grid search (Flaw 4 fix).

---

### Process Flaw 2 — Stage 3 lateral search

**Fixed (Run 8):** Spiral search implemented — 17 XY offsets (center + 4 cardinals × 4 radii at 5/10/15/20 mm), activates after 3 cm no-contact descent, offset locked during consecutive-confirmation steps.

**Run 11 result:** Spiral found contact at (+0,+10mm) offset in T1 retry. Stage 4 descended there and "completed" (pos_error=0.005m) but tier_3=0 — the 10mm offset from calibrated zone centre missed the SFP port aperture (≈3–4mm radius).

**Fix applied (2026-04-06):** After 3-consecutive contact confirmation at a spiral offset, probe 4 cardinal neighbours at ±5mm at contact_z+2mm. Centroid of all positions showing contact is used as Stage 4 XY instead of the first-locked spiral offset. Probing at contact_z+2mm avoids scraping. Falls back to original confirmed position if all neighbours miss.

---

### Process Flaw 3 — Scouting Z unreachable under cable load (RESOLVED — FTS Stage 1)

Run 8 confirmed the Z-plateau problem affects SFP as well as SC:

| Trial | Cable | Commanded Z | Actual plateau | Shortfall |
|-------|-------|-------------|----------------|-----------|
| T1/T2 (Run 8) | SFP | 0.420 | ≈0.374 | 4.6 cm |
| T3 (Run 8)    | SC  | 0.320 | ≈0.275 | 4.6 cm |

**Resolved automatically by FTS Stage 1.** The new algorithm operates at transit_z = `connector_z + 0.08` and probes from `connector_z + 0.04` downward — all below the cable-tension plateau for both cable types (SFP transit z≈0.214, SC transit z≈0.095). The arm descends to probe rather than ascending to a scouting height. Bugs 21 (SFP plateau) and the SC plateau component of Bug 14 are no longer applicable.

---

### Process Flaw 4 — Single-view fallback places arm in free air (FIXED — Run 8 → FTS Stage 1)

The single-view fallback back-projects the scouting pixel to an assumed `connector_z`, giving XY that is **9–13 cm from the calibrated zone center** in every Run 8 trial:

| Trial | Fallback XY | Calibrated zone | Error | Stage 3 result |
|-------|-------------|-----------------|-------|----------------|
| T1 (SFP) | (−0.442, 0.203) | (−0.440, 0.336) | 13.3 cm in Y | arm in free air entire 23 cm descent; max local_delta = 2.67 N |
| T2 (SFP) | (−0.441, 0.268) | (−0.440, 0.336) | 6.8 cm in Y | arm hits board housing 3× at z=0.191–0.207, misses port |
| T3 (SC) | (−0.430, 0.417) | (−0.403, 0.353) | 9.1 cm total | arm in free air; local_delta < 1.3 N throughout |

**Root cause:** The scouting pixel detector finds a spurious feature (wrong scene element or board structure) whose back-projection lands well outside the port neighbourhood. The single-view fallback is currently WORSE than using the calibrated zone directly.

**Fix applied:** Replaced all vision-based Stage 1 (triangulation + single-view fallback) with FTS grid search + percolation. Stage 1 now probes centre + 4 cardinals at 20 mm and 40 mm, classifies each as BOARD or VOID, finds VOID adjacent to BOARD as port aperture, then refines with ±8 mm fine probes. No camera or CV processing used.
